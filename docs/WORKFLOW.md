# AIComicBuilder — 制作工作流（优化版）

> 目标：从已结构化剧本到一整集商业动画成片。

---

## 你的实际工作流（优化后）

```
导入剧本 → 解析角色+上传定妆图 → 进入分集 → 解析分镜
    → ✨ AI过渡增强（新）→ 逐镜生成画面 → 生成视频提示词 → 生成视频
    → 合并导出
```

---

## 详细步骤

### 第一步：导入剧本

在项目页面导入剧本。系统会自动：
- 分析出角色列表
- 识别分集结构

**剧本质量提示：** 场景切换处写清楚过渡动作，而不是硬切。
- ✅ `"…人群渐渐散去，镜头缓缓推向场边稻草垛，两道孩童身影隐约可见"`
- ❌ `"…村民们跳舞。/ 龙渊拉着灵瑶滚进稻草堆"` （直接硬切）

---

### 第二步：解析角色 + 上传定妆图

在"角色"页面：
1. 点击 **解析角色** — AI从剧本提取所有命名角色，生成外貌描述
2. 为每个角色上传 **定妆图**（推荐4视角：正面、四分之三、侧面、背面）

> 定妆图是首帧/尾帧人物一致性的核心依据，不上传则AI随机生成外貌。

**关于角色状态：** 同一角色不同状态（如"龙渊·少年版"vs"龙渊·成年版"）可以分别建角色条目并上传各自定妆图。

---

### 第三步：进入分集 → 解析分镜

1. 进入目标分集（如"第一集，共2分30秒"）
2. 点击 **解析分镜** — 从剧本已结构化的分镜内容提取出每个镜头

每个镜头包含：
- **首帧描述**：开场静帧构图
- **尾帧描述**：结束静帧构图  
- **视频脚本**：喂给视频模型的运动文字
- **运镜**：镜头运动指令

检查解析结果是否完整（尤其是 videoScript 字段，要有具体动作描述，不能是模板套话）。

---

### 第四步：✨ AI过渡增强（关键步骤，生成帧前必做）

**在生成任何帧之前执行此步骤。**

在分镜页面的 **Row 1（解析分镜行）** 最右侧，点击琥珀色的 **"✨ AI过渡增强"** 按钮。它和"解析分镜"并排，因为都是文本准备步骤，在图片生成之前做。

**AI会做什么：**
- 逐对检查相邻分镜的场景类型（群演/大景 vs 命名角色）
- 发现以下情况时自动重写首帧/尾帧描述：
  - 群演镜头 → 主角镜头：让群演尾帧构图引导向角色出现区域
  - 主角镜头 → 新角色镜头：建立新角色的环境感
  - 主角镜头 → 群演镜头：以引导性动作结束
- 不修改视频脚本和运镜，只精调首尾帧描述

**耗时：** 2分30秒（约15个分镜）大约需要 1-3 分钟。

**执行一次即可，改完后再生成帧。**

---

### 第五步：设置生成参数（生成帧之前）

在批量操作区顶部的 **"生成设置"条** 中找到以下两个开关（它们影响所有生成步骤）：

| 开关 | 含义 | 推荐设置 |
|---|---|---|
| **AI增强** | 生成前用AI优化每个prompt（提升质量） | ✅ 建议开启 |
| **独立首帧** | 每个分镜首帧独立生成，不继承上一镜尾帧 | ✅ 默认开启，同场景连续时可手动关闭 |

**"独立首帧"开关说明：**

| 场景 | 推荐设置 | 原因 |
|---|---|---|
| 全集场景切换频繁、大量群演/大景 | **开启** | 避免群演画面传染给主角首帧 |
| 同场景连续镜头（角色一直在场）| **关闭** | 关闭时同角色连续镜头保持像素级连续 |
| 不确定时 | **开启** | 更稳定，不会出错 |

> 注意：即使"独立首帧"关闭，系统也会自动检测"群演→命名角色"切换并独立生成，这是底层自动逻辑，不需要手动干预。

---

### 第六步：逐镜生成画面（首帧 + 尾帧）

逐个点击每个分镜的 **"生成画面"** 按钮。

**生成顺序建议：** 从第1个分镜开始顺序生成，不要跳过。因为后续分镜的首帧可能需要参考前一个分镜的尾帧（如果关闭了"独立首帧"）。

**生成后检查清单：**
- [ ] 尾帧人物与首帧是同一个人
- [ ] 服装与定妆图一致（颜色、款式、配件）
- [ ] 人物比例正常（不会突然变高/变矮）
- [ ] 背景不是纯白/纯色

**发现问题：**
- 只重生尾帧：点分镜右上角 → "重生尾帧"
- 只重生首帧：点分镜右上角 → "重生首帧"
- 两帧都不对：点"重生首尾帧"

---

### 第七步：生成视频提示词

逐镜或批量点击 **"生成视频提示词"**。

AI会根据 videoScript + 角色信息生成专为视频模型优化的提示词。建议在生成视频之前做这一步，不要直接用 videoScript 原文。

---

### 第八步：生成视频

逐镜点击 **"生成视频"**，或用批量视频生成。

**分辨率建议：**
- 先用 `480p` 跑通整集，确认效果
- 满意后切 `720p` 正式渲染

**视频生成后检查：**
- [ ] 运动流畅，没有明显抖动/变形
- [ ] 角色保持一致（没有换脸）
- [ ] 视频不是在首帧或尾帧静止（应该有动作）

---

### 第九步：合并导出

全集分镜视频生成完毕后，点击 **"合并视频"**，ffmpeg 自动拼接为完整集数视频。

---

## 连贯性故障排查

| 现象 | 原因 | 解决方案 |
|---|---|---|
| 群演→主角切换PPT感 | 群演尾帧成了主角首帧 | 开启"独立首帧"；或在生成主角帧前重生该镜首帧 |
| 相邻镜头同一角色外貌不一致 | 定妆图未上传，或尾帧prompt优先级不对 | 上传定妆图；开启AI增强；重生尾帧 |
| 尾帧服装换了 | 模型忽略了定妆图指令 | 重生尾帧（现在prompt已强化"设定图优先于首帧"） |
| 视频里角色比例变形 | 视频模型的局限性 | 调整 videoScript 加入人物比例描述（"龙渊约130cm，略高于灵瑶"） |
| 某集整体风格飘移到写实 | 角色重新解析了但没传visualStyle | 重新解析角色会自动用项目画风，这个已修复 |

|---|---|
| AI增强开关 | 分镜页 → 批量操作区顶部"生成设置"条 |
| 独立首帧开关 | 分镜页 → 批量操作区顶部"生成设置"条 |
| ✨ AI过渡增强 | 分镜页 → Row 1（解析分镜行）→ 最右侧琥珀色按钮 |
| 链式生成（整集） | 分镜页 → ⛓ 链式生成（帧+视频） |
