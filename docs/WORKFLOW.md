# AIComicBuilder — 制作工作流

> 目标：从剧本到一整集商业动画成片。  
> 三个帧相关字段（首帧 / AI 尾帧 / 视频尾帧）的含义与衔接规则见 [ARCHITECTURE-FRAMES.md](./ARCHITECTURE-FRAMES.md)。  
> 文中若出现 **「Plan B」**，仅指该文档描述的**当前帧方案**（相对旧的自动链式、Reference 双轨等），不是 UI 里的功能名。

---

## 工作流总览

```
导入剧本 → 解析角色 + 上传定妆图 → 进入分集 → 解析分镜
    → 生成画面（首帧 + 尾帧）→ 生成视频提示词 → 生成视频 → 合并导出
    → 后期剪辑叠加 BGM
```

---

## 详细步骤

### 第一步：导入剧本

在项目页面导入剧本，系统自动识别角色列表和分集结构。

**剧本质量提示：** 场景切换处写清楚过渡动作，不要硬切。
- ✅ `"…人群渐渐散去，镜头缓缓推向稻草垛，两道孩童身影隐约可见"`
- ❌ `"…村民们跳舞。/ 龙渊拉着灵瑶滚进稻草堆"` （硬切）

**剧本音频字段规范：**
- `【音效】` 写场景级同步音效（火焰噼啪、金属碰撞、脚步声）→ 会传给视频模型生成原生 SFX
- `【背景音】` 写 BGM 情绪注记（"温暖弦乐""悲壮铜管"等）→ **仅供后期参考**，系统不会传给视频模型
- `【videoScript】` **禁止写任何配乐/BGM 描述**（如"配乐响起""弦乐渐强"）——会影响视频模型生成每段独立 BGM，合并后音乐不连贯

---

### 第二步：解析角色 + 上传定妆图

在"角色"页面：
1. 点击 **解析角色** — AI 从剧本提取命名角色，生成外貌描述
2. 为每个角色上传 **定妆图**（推荐正面、四分之三、侧面、背面 4 张）

> 定妆图是人物一致性的核心依据，不上传则 AI 随机生成外貌。

同一角色不同状态（如少年版 vs 成年版）可分别建条目并上传各自定妆图。

---

### 第三步：进入分集 → 解析分镜

1. 进入目标分集（如"第一集，共 2 分 30 秒"）
2. 点击 **解析分镜**

AI 一次调用完成所有工作：按 S 级标准生成每个镜头的首帧描述、尾帧描述、videoScript、运镜、对白，同时在生成过程中自检相邻镜头的场景衔接，发现切换类型不匹配（群演→主角、换场景等）时就地修正首/尾帧描述，保证剪辑流畅。

每个镜头包含：
- **首帧描述**：开场静帧构图（人物位置/表情/光线/情绪）
- **尾帧描述**：结束静帧构图（与下一镜头视觉衔接）
- **videoScript**：Seedance 2.0 格式的运动描述，是视频生成质量的核心
- **运镜**：镜头运动指令

检查解析结果时重点看 videoScript，应有具体人物动作和摄影机运动，不能是模板套话。

**目标时长：** 若设置了分集目标时长，AI 会补充叙事内容（反应镜头、角色细节、环境叙事）使总时长趋近目标，而不是靠拉长单镜 duration 注水。

**覆盖规则：**
- 当前版本**已有生成资产（首帧/尾帧/视频）** → 系统自动新建版本，原版本资产完整保留，无确认弹框
- 当前版本**没有任何生成资产** → 直接覆盖当前版本（节省版本槽位）

**效果不满意：** 直接再点一次「解析分镜」重试，若已有资产会自动新建版本，旧版本可随时切回。

---

### 第四步：设置生成参数

在批量操作区顶部 **"生成设置"** 条中：

| 开关 | 含义 | 推荐 |
|---|---|---|
| **AI 增强** | 生成前用 AI 优化每个 prompt | ✅ 开启 |
| **镜头衔接（视频尾帧）** | 视频成功后，自动把本镜 `cut_point` 直拷为下一镜 `anchor_first`（同集同版本） | 默认关；需要连续落幅时开启 |

**镜头衔接说明：**

- 生成画面时：**不会**自动读上一镜尾帧；弹窗默认「独立生成」，也可手动选他镜参考图（AI 重绘）。
- **参考图选择器**：仅列出**本集当前版本**内其他分镜的 `anchor_first` / `anchor_last_ai` / `cut_point`；跨集衔接用首镜 **「承接上一集尾帧」**（路径直拷，不进选择器）。
- 生成视频后：本镜 **视频尾帧** 写入 `cut_point`。
- **衔接开关关**（默认）：下一镜需点 **「承接上一镜尾帧」** 或参考图选 **上一镜 · 视频尾帧**。
- **衔接开关开**：同集下一镜在视频完成后自动获得上一镜 `cut_point`（路径直拷，非 AI 重绘）。
- **跨集**：第二集首镜点 **「承接上一集尾帧」**（上一集最后一镜的 `cut_point` 优先）。

---

### 第五步：生成画面（anchor_first + anchor_last_ai）

逐个点击分镜的 **「生成画面」**，建议从第 1 镜顺序生成。弹窗可选 **独立生成** 或 **参考图**（本集他镜的首帧 / AI 尾帧 / 视频尾帧 `cut_point`）。

**重新生成逻辑（点击「重新生成帧」）：**
- 已有 `anchor_first` + `anchor_last_ai` → 两帧都重新生成
- 仅有首帧 → 只重生首帧（保持原策略）
- 无帧 → 智能判断（时长/角色/描述决定 `first_only` 或双帧）

**生成后检查：**
- `anchor_last_ai` 人物与 `anchor_first` 一致
- 服装与定妆图一致
- 人物比例正常，背景不是纯白/纯色

**有问题：** 分镜菜单 → 重生首帧 / 重生尾帧 / 重生首尾帧

---

### 第六步：生成视频提示词

**自动（推荐）**：点击 **「生成视频」** 时，若本镜尚无 `videoPrompt`，或首/尾帧已更新（系统比对帧文件指纹），会先自动「看图」生成视频提示词（B2），再调用 Seedance/Kling。

**手动（可选）**：
- 批量点击 **「生成视频提示词」** — 一次处理本集所有已有画面的镜头
- 单镜在需要覆盖自动结果、或调 prompt 后想强制刷新时，再点一次

不必等第六步完成才能生视频；自动路径与按钮共用同一套 vision 精炼逻辑。

---

### 第七步：生成视频

在每张分镜卡上点击 **「生成视频」**（已移除整集批量生成，避免无法按上一镜尾帧顺序衔接）。

- 连续镜头：可先开 **「镜头衔接（视频尾帧）」**，或逐镜 **「承接上一镜尾帧」** 后再生成
- 先用 `480p` 跑通整集确认效果，满意后切 `720p` 正式渲染
- 检查：运动流畅、角色一致、有实际动作

**音频说明：**

每段视频只会生成**场景级原生音效**（打击声、环境音、火焰声等），不会生成跨片段 BGM。这是有意为之——各段独立的 BGM 合并后会硬切，破坏观感。完整 BGM 在第八步后期合成。

| 模型 | 音频行为 |
|---|---|
| Seedance 2.0 | 原生同步音效，对白口型同步；无 BGM |
| Kling | 原生场景音效（`sound: on`）；因 prompt 不含配乐描述，不生成 BGM |
| Jimeng | 场景自然音效 |

**音频分层机制（剧本 → 视频）：**

| 剧本标签 | 存储字段 | 运行时处理 | 最终效果 |
|---|---|---|---|
| `【音效】火焰噼啪声` | `soundEffectNote` | 注入视频 prompt `【音效】...` | 模型在视频中生成原生 SFX |
| `【背景音】温暖弦乐` | `bgmNote` | `stripBgmContent` 从 prompt 中**剔除** | 不传给视频模型，仅供后期选曲参考 |
| `videoScript` 里写"配乐响起" | — | `stripBgmContent` 正则兜底剔除 | 同上，不传给视频模型 |

> **关键设计**：`bgmNote` 精确匹配优先——解析分镜后生成的数据直接用存储的原文字符串精确删除，比正则更可靠。正则作为老数据（历史版本）的兜底。

**群演镜头 · 首帧参考图模式（非旧 Reference 双轨）：**

系统自动识别 **群演镜头**（镜头文本中没有命名角色的镜头，如宏观场景、人群全景、环境镜头）：

- **无需预先生成尾帧** — 群演镜头只需生成首帧即可开始生成视频
- 视频生成自动切换为 **参考图模式**（仅传首帧图，由模型自由发挥运动细节）
- Seedance 生成完成后，返回视频**最后一帧**（`return_last_frame`），保存到本镜 **`cut_point`**（UI「视频尾帧」）
- API **不**回传「视频第一帧」；镜间若要用真实画面衔接，应使用 **`cut_point`（尾帧）**，不是视频首帧
- `cut_point` 默认**不会**写入下一镜 `anchor_first`；开启「镜头衔接（视频尾帧）」后自动直拷，否则手动「承接上一镜尾帧」或参考图选择器

以上逻辑在单镜视频生成中生效。

---

### 第八步：合并导出

全集视频生成完毕后，点击 **"合并视频"**，自动拼接为完整集数视频。

合并时自动处理：
- 各片段音频响度归一化（-14 LUFS），消除片段间音量跳变
- 片段边界淡入 0.5s / 淡出 1s，消除 SFX 硬切感

---

### 第九步：后期叠加 BGM（剪映 / PR / DaVinci）

将合并后的视频导入剪辑软件，叠加一条贯穿全集的 BGM 轨。

**推荐流程：**
1. 导入合并视频
2. 新建音频轨，导入 BGM 文件
3. 按 `【背景音】` 注记的情绪描述（可在分镜页查看每镜的 BGM 注记）选曲/剪辑
4. 调整音量（BGM 建议 -18 到 -24 dB，SFX 层保持 -12 dB 左右）
5. 在场景切换处做 BGM 淡入淡出

> `【背景音】` 字段里写的情绪描述（如"温暖木管，轻拨弦乐"）是后期选曲的参考，不会被传给视频生成模型。

## 编剧字段检查清单（S 级 · 与 ARCHITECTURE-PROMPTS 一致）

| 字段 | 写什么 | 不要写什么 |
|------|--------|------------|
| **场景描述（prompt）** | 镜头情节、场次氛围、可能发生的事件 | 不要写适合画进静帧的起止姿态（那是首尾帧的事） |
| **首帧描述** | 动作**开始前**的静止画面 | 不要写运动过程、不要与 prompt 里的进行态矛盾 |
| **尾帧描述** | 动作**结束后**的静止收束 | 必须与首帧不同；不要重复整段运镜链 |
| **videoScript** | 运动散文（角色+单一动作+运镜+一种感官细节） | 不要写 BGM/配乐（用【背景音】注记） |

解析分镜或「重新生成文本」后，若首帧应是空镜/静谧态，请把动态情节只留在 prompt，不要写进首帧描述。

**旧版本分镜数据**：若首尾帧与场景描述明显矛盾（例如 prompt 写「魔族入侵中」、首帧写「静谧小镇」），需在本集再点 **「解析分镜」**（有资产会新建版本）或单镜 **「重新生成文本」**，组装层不会自动改正史字段。

## 单镜头修改操作速查

| 想做什么 | 点哪个按钮 | 注意事项 |
|---|---|---|
| 重写 videoScript / 首尾帧描述 | **重新生成文本** | 不会覆盖场景描述（prompt）；不会删除已生成的帧/视频 |
| 恢复到原始剧本文本 | **从剧本还原** | 只恢复文本字段，帧/视频不受影响 |
| 用上一镜视频尾帧当本镜首帧 | **承接上一镜尾帧**（本集） | 直拷 `cut_point`，跳过 Seedream |
| 用上一集最后一镜尾帧当本镜首帧 | **承接上一集尾帧**（跨集首镜） | 直拷上一集最后一镜 `cut_point` |
| 连续多镜自动衔接 | 开启 **镜头衔接（视频尾帧）** | 每镜视频完成后自动写下一镜 `anchor_first` |
| 用任意他镜画面参考生成首帧 | **选择参考图生成** | AI 重绘，非直拷 |
| 重新生成首帧 | 菜单 → **重生首帧** | 弹出参考图选择器 |
| 重新生成尾帧 | 菜单 → **重生尾帧** | 需要首帧存在 |
| 重新生成首帧+尾帧 | 菜单 → **重生首尾帧** | 若之前只有首帧，只重生首帧 |

## 故障排查

| 现象 | 原因 | 解决方案 |
|---|---|---|
| 生成的单段视频有背景音乐 | 旧版本的 videoPrompt 含 BGM 描述；新解析的分镜 `bgmNote` 未入库 | 点"**从剧本还原**"重新解析分镜（重建 `bgmNote` 字段），再点"**重新生成文本**"，然后重新生成视频 |
| 合并视频背景音乐硬切 | videoScript 里写了配乐描述，模型为每段生成 BGM | 点"重新生成文本"（已内置 BGM 禁用规则）；或点"从剧本还原"重新解析 |
| 群演→主角切换有 PPT 感 | 误把群演 `cut_point` 当下镜首帧 | 不要点「承接上一镜尾帧」；用参考图选主角镜或独立生成 |
| 第二集接不上第一集结尾 | 未点「承接上一集尾帧」 | 第二集第 1 镜点该按钮，或手动参考图（仅本集列表） |
| 视频 ENOENT / 误用首尾帧模式 | DB 有帧路径但磁盘文件已删 | 分镜卡红框「文件缺失」→ 重生对应帧；系统不自动清 DB |
| 相邻镜头同角色外貌不一致 | 定妆图未上传 | 上传定妆图；开启 AI 增强；重生尾帧 |
| 尾帧服装换了 | 模型忽略定妆图指令 | 重生尾帧 |
| 视频里角色比例变形 | 视频模型局限 | 在 videoScript 加人物比例描述 |
| 整集风格飘移到写实 | visualStyle 未注入 | 已修复，重新解析角色即可 |
| 解析后总时长不足 | LLM 受剧本镜头数限制，未插入补充镜头 | 重新点「解析分镜」重试 |
| 相邻镜头场景切换割裂 | LLM 未正确执行过渡规则 | 重新点「解析分镜」重试 |
| 帧图片刷新后没变化 | 浏览器缓存 | 已修复（文件名带时间戳），强刷页面 |

## 功能位置速查

| 功能 | 位置 |
|---|---|
| **看板视图（Kanban）** | 分镜页标题右侧 → 点 **看板**（四宫格）；四列按进度归类，**点卡片**打开右侧 **抽屉**（详情工作区） |
| **列表视图** | 点 **列表**；每镜一张 `ShotCard`，也可点开同一抽屉 |
| **抽屉（详情）** | 看板点卡片 / 列表紧凑模式；含生成画面（含参考图/承接）、三格帧上传与单帧重生、视频与版本历史 |
| **远程视频恢复提示** | 列表与抽屉视频区：天蓝标签「可恢复远程结果」→ 再点「生成视频」优先下载远程成片 |
| **画质增强 ↑720p** | 列表与抽屉：已有成片且非 720p 时显示（需 MediaKit 配置） |
| **列表额外能力** | 各字段 AI 优化（抽屉无） |
| 解析分镜 | 分镜页 → Row 1 → 解析分镜 |
| AI 增强 | 分镜页 → 批量操作区顶部「生成设置」 |
| 版本管理 | 分镜页顶部 → 新建版本 / 切换版本 / 删除版本 |
| 单镜重写文本 | 分镜卡片 → 重新生成文本 |
| 单镜还原剧本 | 分镜卡片 → 从剧本还原 |
